# 01 - 开发环境配置

> 🎯 **学习目标**：搭建现代化的 Web 开发环境，安装并配置必要的开发工具。

---

## 一、Visual Studio Code 安装

### 1.1 什么是 VS Code

Visual Studio Code（简称 VS Code）是由微软开发的免费开源代码编辑器，是目前最受欢迎的 Web 开发编辑器。它不仅是"高级文本编辑器"，更是集成了语法高亮、智能补全、调试工具、Git 集成等功能的完整开发环境。

### 1.2 核心功能

| 功能 | 说明 |
|:---|:---|
| **Syntax Highlighting** | 语法高亮，用不同颜色显示代码，提高可读性 |
| **Auto-completion** | 自动补全，输入时智能提示代码，减少打字错误 |
| **Debugging Tools** | 调试工具，设置断点、逐步执行、查看变量值 |
| **Extensions** | 扩展系统，通过插件增强功能，支持各种语言和框架 |
| **Built-in Terminal** | 内置终端，无需切换窗口即可执行命令 |
| **Git Integration** | Git 集成，可视化地进行版本控制操作 |

### 1.3 安装步骤

1. 访问官网：https://code.visualstudio.com
2. 下载对应系统的安装包（Windows/Mac/Linux）
3. 运行安装程序，建议勾选以下选项：
   - ☑️ "Add to PATH"（添加到环境变量）
   - ☑️ "Register Code as an editor for supported file types"
   - ☑️ "Add 'Open with Code' action to Windows Explorer"

### 1.4 必装扩展推荐

```
📦 开发效率类
├── Live Server          # 本地开发服务器，保存自动刷新
├── Prettier            # 代码格式化，保持风格一致
├── GitHub Copilot      # AI 代码助手
└── Code Spell Checker  # 拼写检查
```

安装方法：点击左侧 Extensions 图标（或按 `Ctrl+Shift+X`），搜索名称后点击 Install。

## 二、Git 安装与配置

### 2.1 什么是 Git

Git 是一个**分布式版本控制系统**，用于跟踪代码的变更历史。它让你可以：
- 保存代码的每个版本（像游戏的存档点）
- 查看何时、何人修改了什么内容
- 回滚到之前的版本
- 多人协作而不互相覆盖工作

### 2.2 安装步骤

1. 访问官网：https://git-scm.com
2. 下载对应系统的安装包
3. 运行安装程序，使用默认选项即可

### 2.3 验证安装

打开终端（VS Code 中按 `Ctrl+` `），输入：

```bash
git --version
```

预期输出：`git version 2.xx.x`

### 2.4 初始配置

首次使用 Git 需要配置用户信息（只需配置一次）：

```bash
# 设置用户名（会显示在提交记录中）
git config --global user.name "Your Name"

# 设置邮箱（建议与 GitHub 账号一致）
git config --global user.email "your.email@example.com"

# 验证配置
git config --list
```

### 2.5 配置说明

- `--global` 表示全局配置，适用于所有项目
- 用户名和邮箱会出现在每次提交记录中
- 建议使用真实信息，方便团队协作时识别

## 三、Node.js 安装

### 3.1 什么是 Node.js

Node.js 是一个让 JavaScript 可以在**浏览器之外运行**的运行时环境。它使开发者能够：
- 在服务器端运行 JavaScript
- 使用 npm 管理项目依赖
- 运行构建工具和开发服务器

### 3.2 安装步骤

1. 访问官网：https://nodejs.org
2. 下载 **LTS（长期支持）版本**（推荐稳定版）
3. 运行安装程序，按提示完成

### 3.3 验证安装

```bash
# 检查 Node.js
node --version
# 预期输出：v20.x.x 或更高

# 检查 npm（Node Package Manager）
npm --version
# 预期输出：10.x.x 或更高
```

### 3.4 npm 的作用

npm（Node Package Manager）是随 Node.js 一起安装的包管理工具，用于：
- 安装第三方库（如 React、Vue、lodash 等）
- 管理项目依赖
- 运行脚本命令

```bash
# 示例：安装开发工具
npm install --save-dev vite

# 示例：初始化新项目
npm init -y
```

## 四、浏览器开发者工具

### 4.1 推荐的浏览器

| 浏览器 | 特点 |
|:---|:---|
| **Chrome** | DevTools 功能最全面，文档丰富 |
| **Edge** | 基于 Chromium，内置优秀辅助功能工具 |
| **Firefox** | CSS Grid 和可访问性工具出色 |

### 4.2 打开 DevTools

- 快捷键：`F12` 或 `Ctrl+Shift+I`
- 右键页面 → "检查"（Inspect）

### 4.3 主要面板

| 面板 | 功能 |
|:---|:---|
| **Elements** | 查看和实时编辑 HTML/CSS |
| **Console** | 查看日志、执行 JavaScript |
| **Network** | 监控网络请求和加载性能 |
| **Sources** | 调试 JavaScript 代码 |
| **Application** | 查看 LocalStorage、Cookie 等 |

## 五、开发环境检查清单

完成安装后，在终端中运行以下命令验证：

```bash
# 1. 检查 VS Code（需添加到 PATH）
code --version

# 2. 检查 Git
git --version

# 3. 检查 Node.js
node --version

# 4. 检查 npm
npm --version
```

所有命令都应返回版本号，无错误信息。
