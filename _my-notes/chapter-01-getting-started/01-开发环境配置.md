# 01 - 开发环境配置

> 🎯 **学习目标**：搭建现代化的 Web 开发环境，安装并配置必要的开发工具。

---

## 🛠️ 所需工具清单

| 工具 | 作用（一句话） | 官方下载 |
|:---|:---|:---|
| **VS Code** | 代码编辑器，提供语法高亮、自动补全、调试等功能 | https://code.visualstudio.com |
| **Git** | 版本控制系统，记录代码变更历史，支持团队协作 | https://git-scm.com |
| **Node.js** | JavaScript 运行时环境，让 JS 可以在浏览器外运行 | https://nodejs.org |
| **浏览器** | 预览和调试网页，推荐 Chrome / Edge / Firefox | 已预装 |

---

## 🔍 安装验证命令

打开终端（Terminal/命令提示符），输入以下命令检查是否安装成功：

```bash
# 检查 VS Code（如果已添加到 PATH）
code --version
# 预期输出：1.xx.x (版本号)

# 检查 Git
git --version
# 预期输出：git version 2.xx.x

# 检查 Node.js
node --version
# 预期输出：v20.x.x (LTS 版本)

# 检查 npm（Node 包管理器）
npm --version
# 预期输出：10.x.x
```

---

## ⚙️ Git 初始配置

首次使用 Git 需要配置用户信息：

```bash
# 设置用户名（会显示在提交记录中）
git config --global user.name "Your Name"

# 设置邮箱（建议与 GitHub 账号一致）
git config --global user.email "your.email@example.com"

# 验证配置
git config --list
```

## 💡 VS Code 推荐扩展

| 扩展名 | 作用 |
|:---|:---|
| **Live Server** | 本地开发服务器，保存自动刷新浏览器 |
| **GitHub Copilot** | AI 代码助手，提供智能代码建议 |
| **Prettier** | 代码格式化工具，保持代码风格一致 |
| **Code Spell Checker** | 拼写检查，减少代码中的拼写错误 |

安装方法：VS Code → 扩展（Ctrl+Shift+X）→ 搜索名称 → 点击安装

---

## 🌐 浏览器开发者工具

按 `F12` 或右键选择"检查"，可以打开 DevTools：

| 面板 | 功能 |
|:---|:---|
| **Elements** | 查看和编辑 HTML/CSS |
| **Console** | 查看错误信息、执行 JS 代码 |
| **Network** | 监控网络请求和加载性能 |
| **Sources** | 调试 JavaScript |
